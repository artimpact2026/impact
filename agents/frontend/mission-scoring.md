# 미션 점수 산정 방식 (v2 — 깊이 있는 대화 + 적합도 추적)

이 문서는 21개 미션(공통 9 + 지역 12)에서 사용자가 답하는 선택지에 따라 **축적 점수**와 **적합도**가 어떻게 계산되는지 설명합니다.

---

## 1. 두 가지 점수

| 점수 | 무엇 | 어떻게 |
|------|------|------|
| **축적 점수** | 이 지역에서 얼마나 많이 체험했는지 | 미션 완료 시 mission.reward 가산 (예: 병원 +5, 관계형성 +15) |
| **적합도** | 이 지역이 나에게 얼마나 맞는지 (0~100) | 라이프스타일 베이스 + 미션 답변 누적 + 미션 완료 보너스 |

---

## 2. 축적 점수 (`mission.reward`)

각 미션의 reward 값은 **카테고리 + 난이도** 기반으로 다음 정책을 따랐습니다.

| 미션 분류 | 점수 |
|----------|------|
| 짧은 정보형(병원/시장/생활비/교통/식비) | **+5** |
| 의미 있는 체험(루틴) | **+8** |
| 관계 형성(가게/이주민/지역 관계 미션) | **+15** |
| 감정·분위기형 (지역 풍경) | **+10** |
| 우편함 (편지 1장) | **+3** |
| 최종 리포트 | **+100** |

지역 미션도 같은 정책으로 +5 / +8 / +10 / +15 중 선택.

---

## 3. 적합도 (`fitScore` + `calculateMatch`)

### 3-1. 베이스
- 사용자 라이프스타일이 `residence.matchType`과 **같으면 70**
- 다르면 **50**

### 3-2. 미션 답변에 따른 누적 (`fitScore`)

옵션마다 `traits: LifeStyleType[]`을 부착했어요. 옵션을 고르면 다음 공식으로 가감:

```ts
if (option.traits[0] === residence.matchType) → +2
else if (option.traits.includes(residence.matchType)) → +1
else → 0
```

즉, **답변이 그 지역의 추천 유형을 정확히 반영하면 +2점**, 보조적으로 반영하면 +1점, 무관하면 0점.

음수 페널티는 두지 않았어요. 답을 어느 쪽으로 해도 사용자를 평가하지 않고, 잘 맞는 것만 가산하는 톤.

### 3-3. 보너스
- 완료한 미션 수 × 1점 (최대 +8)

### 3-4. 최종 식

```
match = clamp(0, 100, base + fitScore + min(8, completedCount))
```

### 3-5. 라이프스타일 → 지역 매칭

| 거주지 | matchType | 우선 가산되는 답변 패턴 |
|--------|----------|------|
| 강화도 | 자연탐험형 | 자연/느림/조용함 |
| 광양 | 디지털노마드형 | 작업/효율/일과 쉼의 균형 |
| 거제도 | 레저형 | 활동/바다/사람과 어울림 |

---

## 4. 미션별 대화 깊이 (turn 수)

기존 2~3 turn에서 **3~6 turn으로 확장**. 분기(branching) + 합류(converging) 구조로 사용자가 같은 미션을 다시 해도 다른 길로 갈 수 있게 했습니다.

| 미션 | turn 수 | 비고 |
|------|--------|------|
| 병원 접근성 | 6 | 진입(병원/산책) → 이동수단 분기 → 응급/일상 분기 → 마무리 |
| 전통시장 물가 | 4 | 가격 안내 → 깊이 / 시간대 분기 → 한 달 식비 시뮬레이션 |
| 생활비 시뮬레이션 | 4 | 비용 구조 → 절약 vs 여유 / 차 유지비 분기 |
| 교통 접근성 | 4 | 막차/차 없음 / 차 보유 분기 |
| 하루 루틴 | 4 | 아침 → 점심 → 저녁 → 일상 리듬 평가 |
| 하루 식비 기록 | 3 | 안내 → 수치 입력 → 비교 |
| 동네 가게 들르기 | 4 | 단골/추천 / 사람과의 거리감 |
| 이주민 만나기 | 5 | 적응기 / 현재 만족도 / 후회 분기 |
| 우편함 | 1 | 모달로 진행 (시뮬레이션 외) |
| 지역 미션 12개 | 평균 3~4 |  |

---

## 5. NPC가 알려주는 "현실 정보"

스마트폰 안에서 시뮬레이션만 가능하므로, 모든 현실 정보(물가/거리/배차/기간 등)는 **NPC 말풍선**으로 제시했습니다. 사용자는 입력하지 않고 듣고/판단만 합니다.

예시:
- 병원: "도보 12분, 850m"
- 시장: "오이 한 박스 6,000원, 시금치 한 단 1,500원"
- 생활비: "월세 25~40만원, 식비 25~35만원, 합쳐서 약 80만원선"
- 교통: "버스 평균 배차 35분, 막차 21시 30분"
- 카페: "한 잔 평균 5,000원, 와이파이 빠름"

**유일하게 입력받는 케이스**: 하루 식비 기록 (mode: 'numeric'). 입력값을 평균(9,500원)과 비교해 절약형/보통/풍족형으로 분기.

---

## 6. 핵심 흐름

```
미션 리스트
   ↓
mode === 'mailbox': MailboxModal (편지 카드)
mode === 'map-dialogue': MissionTraveling (1.5s) → MissionExecute (말풍선+선택지)
mode === 'map-info':     MissionTraveling (1.5s) → MissionExecute (정보형 짧은 대화)
mode === 'dialogue':     MissionExecute 바로
mode === 'numeric':      MissionExecute (도중 수치 입력)
   ↓
옵션마다 fitDelta 누적
   ↓
마지막 옵션 → onComplete(누적 fitDelta) → regionProgress.fitScore 갱신
   ↓
미션 리스트 복귀 (모두 완료 시 daily-summary)
```

---

## 7. 향후 보완 여지

- 옵션에 직접 `fitDelta: number` 명시(traits 대신) — 더 세밀한 가감
- 미션별 시간대(아침/저녁) 설정 → 같은 미션도 다른 대화
- 미션 다시하기 시 첫 답변과 다른 답변 비교
- 우편함을 진짜 "매일 1장" 시스템으로 (시간 기반)
