# 지역별 미션 정리 (8개 추천 레지던스 × 각 4개 = 32개)

각 레지던스마다 라이프스타일 유형에 맞는 지역 특색 미션 4개씩.
공통 미션 9개와 합쳐 사용자가 선택한 지역에서는 총 **13개 미션**을 체험.

진행 방식 표기:
- 🗺️ `map-info` — 지도 이동 후 정보형 안내 (1~3 turn)
- 🗣️ `map-dialogue` — 지도 이동 후 NPC와 대화 (3~5 turn)
- 💬 `dialogue` — 바로 대화 (3~5 turn)
- 🔢 `numeric` — 대화 + 수치 입력 단계
- 📮 `mailbox` — 우편함 모달

분류 표기:
- 💡 생활현실형
- 🤝 관계형성형
- 🌿 감정/분위기형

## 🧮 적합도(fit) 가산 패턴

각 옵션의 `traits` 배열로 가산점 결정:
- `traits[0] === residence.matchType` → **+2**
- `traits.includes(residence.matchType)` → **+1**
- 어긋남/비어 있음 → **0** (감점 없음)

지역 미션의 `traits` 분포:
- 거주지 유형(matchType)이 1순위로 잘 나오게 작성
- 보조 유형은 2순위에 배치해 사용자가 다른 답을 골라도 +1을 받게 함
- 예: 강화도(자연탐험형) 미션의 옵션에 `traits: ["자연탐험형", "집돌이형"]` 패턴이 자주 등장 — 자연탐험형 +2, 집돌이형은 +0이지만 traits[0]에 들어가면 +2

상세 점수 산정은 `agents/frontend/mission-scoring.md` 참고.

## 🔧 데이터 위치

- 공통 미션 (9개): `frontend/src/data/missions.ts` → `commonMissions`
- 지역 미션 (32개): `frontend/src/data/regionMissions.ts` → `regionMissions[residenceId]`
- 통합 함수: `missionsForResidence(residenceId)` → 공통 + 지역 미션 합본

레지던스가 추가될 때마다 `regionMissions` 객체에 매핑 + 4개 미션 작성. 매핑 키는 `residence.id`와 일치해야 함.
