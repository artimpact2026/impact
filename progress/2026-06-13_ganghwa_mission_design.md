- 컨셉: "미션을 하나씩 따라가다 보면 어느새 달라져 있는 나의 일상을 발견한다"

**출처:**

- 인증 사례 인용: 페이지에 게시된 "강화도에서 하타요가를 할 수 있다니" 같은 실 사용자 후기 (날짜 기준 2026.04.27, 04.24 등)

### 1-3. 우리가 강화도를 선택한 이유

1. **도구가 실재한다** — 청풍의 잠시섬이 강화에서 실제로 운영되는 프로그램이고, 우리 시뮬레이션의 모든 미션이 이 실재 공간/사람/활동에 anchor 됨.
2. **수도권이지만 정주인구 감소 지역** — 강화군은 인천광역시 소속(수도권 정비계획법 기준 수도권)이나 행정구역상 군 단위이고 인구 감소·고령화 지역이라, '지역살이/관계인구' 담론에 자연스럽게 들어맞음. 이게 "지방"이라는 용어를 우리가 의도적으로 안 쓰는 이유 (수도권에도 관계인구가 필요함).
3. **실재 인용 가능한 인물·후기·미션 패턴** — 잠시섬 18기 후기에 등장하는 텃밭 클럽, AR 미션 앱, 순무김치, 마당뷰 등 우리 미션의 직접 근거가 됨.

## 2. 미션 설계 원리

### 2-1. 3박 4일 구조의 이유

- **체류 일수 = 4**: 잠시섬 19기는 실제로 5박이지만, 우리 시뮬레이션은 데모 페이스 위해 3박 4일로 압축.
- **하루 단위 서사 아크**:

| Day | 테마 | 의도 |
|---|---|---|
| 1 | **첫인상** (도착·바다·동네) | 도시 톤에서 강화 톤으로 전환. 작은 첫 접촉. |
| 2 | **관계** (사람 만나기) | 지역 공동체와의 결을 직접 경험. |
| 3 | **생업** (손으로 만들기) | "여기서 무엇을 하며 살까"의 감각. |
| 4 | **인프라·정착 상상·작별** | 현실 점검 + 정서적 클로징. |

이 4일 아크는 청풍의 이주 단계 프레임 **"탐색 → 이주(살아보기) → 정착"** 을 하루 단위로 압축한 형태.

### 2-2. 시간대 분할 (아침 / 낮 / 저녁)

각 일차에 시간대별로 미션 1개씩 배치. 이유:
- 관광이 아닌 **생활의 결**을 인지시키려면 하루의 리듬 체험이 필수
- 시간대마다 동네가 다른 표정 (아침 = 정적·책방·시장 / 낮 = 활동·갯벌·체험 / 저녁 = 관계·식사·노을)

### 2-3. 필수 12개 + 보너스 9개

- **필수**: 4일 × 3시간대 = 12개. 무조건 거쳐야 하는 메인 동선.
- **보너스**: 9개. 자유 탐색·추가 깊이. 시간대·일차 종속 없음.
- **종료 조건**: `필수 3개 (그날의 아침·낮·저녁) AND 보너스 2개 이상 누적` 충족 시 하루 종료 의식 작동.
  - 코드: `frontend/src/hooks/useDayComplete.ts`
  - 상수: `DAY_COMPLETE_BONUS_REQUIRED = 2`, `URGENCY_THRESHOLD = 2`
  - 사용자는 "오늘 하루를 잘 보내려면 보너스도 한두 개 챙겨야 한다"는 감각을 갖게 됨 → 추가 깊이로 자연스럽게 유도

**실재 근거:**
- **함민복 시인** — 강화에서 활동하는 시인. 갯벌과 강화 바다를 노래한 시로 유명. 미션 대화에서 "함민복 시인이 여기 갯벌을 시로 썼다"고 NPC가 직접 인용.

- 강화 실재 동네 책방: **국자와주걱**(양도면 도장리), **책방시점**, **딸기책방**. 우리 미션엔 특정 상호명을 박지 않았지만, 위 책방들이 함민복 시인·김중미 작가 등 강화 로컬 작가와 함께 북토크 프로그램을 운영하는 ecosystem이 실재함.

**출처:**
- 함민복 시인 갯벌 시 관련 — 위키백과 / 한국문학 데이터베이스 등에서 직접 검증 가능
- 국자와주걱 책방·강화 작가 북토크 — 강화군 책방 행사 기록, 책방 SNS 등 ('동네 책방과 동네 작가가 함께하는 문학여행' 프로그램)

**연결 의도:**
- → Day 1 낮 **갯벌 미션과 정서 연결**. 책방지기가 "오늘 낮에 갯벌 가시면, 시 한 편 머릿속에 두고 가보세요. 풍경이 달라져요" 라고 명시. **첫날 = 강화의 바다**라는 한 주제로 묶이게 함.

**발표 무기:** 특정 상호명 X (광고처럼 안 보임), 그러나 NPC 대사에 **실재 강화 시인 이름**을 자연스럽게 박아 신뢰도 ↑.

## 6. 정서적 연결망 — Cross-Day 동선

미션끼리 의도된 연결 (NPC 대사로 명시):

```
Day 1 아침 (책방, 함민복 갯벌 시)
   ↓ "오늘 낮에 갯벌 가시면 시 한 편 머릿속에 두고 가보세요"
Day 1 낮 (갯벌 걷기)
   ↓ "사흘 다니면 몸이 좀 무거워질 때"
Day 4 낮 (온천, 정착 상상)
   ↓ "오늘 저녁 일몰 보러 가신댔죠?"
Day 4 저녁 (일몰, 작별)


Day 3 아침 (소창 짜기 "손으로 만드는 감각")
   ↓
Day 3 저녁 (방명록 "오늘 만든 것 기록")


Day 1 아침 (책방 "오늘 만난 한 문장")
   ↓ (모티프 회수)
Day 3 저녁 (방명록 "한 줄 기록")
```

이 연결망 덕분에 사용자는 미션 하나하나가 독립적이 아니라 **하나의 시간 흐름**으로 인지하게 됨. 발표에서 "각 미션이 어떻게 엮였는가" 질문 받으면 이 그림으로 답 가능.

## 7. 발표 시 주의 사항

### 7-1. 광고처럼 도배 X
- 특정 상호명 (스트롱파이어, 아삭아삭순무민박, 국자와주걱 등) 은 미션 제목·설명에 박지 않음.
- NPC 라벨/대사에서 결만 자연스럽게 노출 ("사랑방 펍", "마당뷰 게스트하우스").
- 대표님 입장에서 "왜 우리 상호만 콕 짚어?" 라는 의문 회피.

### 7-2. 검증 가능한 사실만 인용
- 고려 천도 (1232년) — 역사적 사실
- 강화 갯벌 = 세계 3대·람사르 — 공식 등재
- 함민복 시인 = 강화 활동 시인 — 공개 정보
- 잠시섬 미션 인증 시스템 — 강화유니버스 공식 사이트에 직접 노출

### 7-3. 의심받기 쉬운 부분은 명시 안 함
- 잠시섬 **기수 번호**: 사용자 prompt 기준 18기 (실제는 19기 모집 중) — UI에는 기수 명시 안 함. "잠시섬"만.
- 5박 8만원 같은 정확한 가격 — UI에는 사용 안 함.
- 특정 인물명 (대표님 외) — NPC는 직책으로만 표현.

### 7-4. "왜 이 미션을 골랐냐" 질문 대비
각 미션마다 위 표의 "실재 근거 + 출처" 두 줄을 외워두면 즉답 가능. 코드 파일 (`regionMissions.ts`, `ganghwaStory.ts`) 도 그대로 보여줄 수 있음.

## 부록 A. 잠시섬 핵심 카피 (인용 가능)

발표 슬라이드에 직접 인용해도 되는 청풍 공식 카피들:

- **"환대로 만들어가는 세계"** (메인 슬로건)
- **"잠시 떠나온 곳이 내 동네처럼, 낯선 사람이 다시 만나고 싶은 친구가 되어가는 일"**
- **"강화에서 잠시 다른 삶에 접속할 시간"** (여름 캠페인)

이 세 줄은 우리 미션 설계 철학과 정확히 호응함:
- "환대" — NPC 톤의 기본 키워드
- "잠시" — 시뮬레이션의 시간 감각
- "다른 삶에 접속" — 도시 vs 강화 결의 대비
